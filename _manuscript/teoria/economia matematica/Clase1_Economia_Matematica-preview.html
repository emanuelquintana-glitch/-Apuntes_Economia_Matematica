0.1 ¿Qué es la Economía Matemática?

La economía matemática no es una rama especializada de la economía como las finanzas públicas o el comercio internacional, sino más bien un método de análisis económico donde el economista utiliza:

  • Símbolos matemáticos para enunciar problemas
  • Teoremas matemáticos para facilitar el razonamiento
  • Técnicas avanzadas como álgebra matricial, cálculo, ecuaciones diferenciales

1 La Naturaleza del Análisis Económico: Matemática vs. Literaria

El entendimiento de la Economía Matemática comienza por diferenciarla de la Economía “Literaria” (o no matemática). Esta distinción no radica en el tema de estudio, sino en el instrumental lógico y el método de análisis empleado.

1.1 Diferencias Fundamentales en el Método de Análisis

Aspecto Clave Economía Matemática (Análisis Formal) Economía Literaria (Análisis Intuitivo)
Lenguaje y Comunicación Se basa en el uso de símbolos, variables y ecuaciones (\(\frac{\partial U}{\partial x_i} = \lambda p_i\)). La comunicación es universal y concisa. Se fundamenta en el uso de palabras, enunciados y descripciones narrativas. La interpretación puede variar.
Estructura Lógica Utiliza la lógica deductiva formal basada en Teoremas, Axiomas y Proposiciones (ej. \(A \implies B\)). La conclusión es inevitable si las premisas son verdaderas. Emplea la lógica literaria o verbal, que a menudo se apoya en la inducción, la analogía y la persuasión retórica.
Precisión y Rigor Precisión es alta y cuantificable. El razonamiento es explícito y susceptible de ser verificado paso a paso. Se buscan condiciones necesarias y suficientes. Precisión es variable. El rigor depende de la habilidad del autor para articular las ideas. Se centra en la viabilidad conceptual y la plausibilidad.
Suposiciones (Hipótesis) Explícitas y formalizadas en cada etapa del modelo (ej. convexidad de preferencias, rendimientos decrecientes). Esto permite el Test de Sensibilidad. Pueden ser implícitas o diluidas en el texto. Esto dificulta la identificación precisa de los límites de validez del argumento.
Naturaleza de la Conclusión Conclusión cualitativa y cuantitativa (ej. se puede resolver para un valor específico o demostrar la existencia de un equilibrio). Conclusión predominantemente cualitativa, enfocada en la dirección de los efectos o la relación causal general.

1.2 Implicaciones Académicas

La Economía Matemática se distingue por su capacidad para:

  1. Garantizar la Coherencia Interna: Si un economista formula un modelo de forma matemática, el uso de teoremas garantiza que las conclusiones derivadas se desprendan lógicamente de las suposiciones iniciales.
  2. Facilitar la Aplicación Empírica: Al ser los modelos explícitos y cuantificados, la traducción a modelos econométricos para la contrastación estadística resulta directa.
  3. Abordar la Complejidad: Permite analizar sistemas con múltiples variables e interdependencias que serían intratables o propensos a errores en la lógica verbal.

import numpy as np
import matplotlib.pyplot as plt

# -------------------------------
# Funciones de utilidad Cobb-Douglas
# -------------------------------

def utilidad_general(bienes, alpha=0.5):
    """
    Función de utilidad Cobb-Douglas generalizada:
    U(x₁, x₂, ..., xₙ) = ∏ xᵢ^α
    """
    return np.prod(bienes**alpha)

def utilidad_2d(x, y, alpha=0.5):
    """
    Cobb-Douglas en 2 bienes: U(x, y) = x^α * y^α
    """
    return (x**alpha) * (y**alpha)

# -------------------------------
# Cálculos de ejemplo
# -------------------------------

bienes_2d = np.array([2, 3])

1.3 Economía Matemática vs. Econometría: Diferencias Clave

La distinción entre Economía Matemática y Econometría reside fundamentalmente en su objetivo primario y su método de validación. Ambas son herramientas formales, pero abordan el análisis económico desde ángulos opuestos:

1.3.1 1. Economía Matemática (La Lógica Deductiva)

Se enfoca en la formulación abstracta y el análisis teórico de los modelos económicos.

Aspecto Foco
Objetivo Primario Establecer la coherencia interna y la validez lógica de las teorías.
Método Principal Deducción Matemática. Parte de un conjunto de supuestos (\(A_1, A_2, \ldots\)) para demostrar la existencia o propiedades de un equilibrio.
Enfoque Teórico y Analítico. Se centra en las condiciones necesarias y suficientes para que ocurra un fenómeno.
Producto Final Un Teorema o una Proposición formalmente demostrada.

1.3.2 2. Econometría (La Evidencia Empírica)

Es el puente entre la teoría económica y la realidad empírica. Su propósito es contrastar las teorías y cuantificar las relaciones usando datos reales.

Aspecto Foco
Objetivo Primario Proporcionar validez empírica a las teorías y estimar relaciones causales.
Método Principal Inducción Estadística. Parte de la observación de datos muestrales para inferir y probar hipótesis sobre la población (ej., estimar el valor de un coeficiente \(\beta\)).
Enfoque Empírico y Cuantitativo. Utiliza métodos estadísticos para manejar la estocasticidad (aleatoriedad) inherente a los datos económicos.
Producto Final Una Estimación (un valor numérico) y sus estadísticos de prueba (valores \(t\) o \(p\)).

1.3.3 Síntesis: Complementariedad

La Economía Matemática provee el “esqueleto” lógico de la teoría (el modelo \(Y=f(X)\)), mientras que la Econometría utiliza ese esqueleto para agregar la “carne” de la realidad (la estimación \(\hat{Y} = \hat{\beta}X + \varepsilon\)) y determinar si el modelo es plausible en la práctica.

2 Aplicación Práctica: De la Teoría a la Implementación

2.1 Fundamento teórico detrás del modelo

El modelo se basa en la teoría de la elección del consumidor en microeconomía, específicamente en la maximización de la utilidad sujeta a una restricción presupuestaria.

2.1.1 El Problema de Maximización

El consumidor busca la combinación de bienes \(\mathbf{x} = (x_1, x_2, \ldots, x_n)\) que le proporciona la mayor satisfacción (utilidad) dado su ingreso limitado y los precios de los bienes.

Función de Utilidad (Cobb-Douglas Logarítmica)

La función de utilidad utilizada es una forma logarítmica de la función Cobb-Douglas, que se caracteriza por ser homotética y cóncava, garantizando la existencia de una solución única y la disminución de la utilidad marginal.

\[U(\mathbf{x}) = \sum_{i=1}^n \alpha_i \ln(x_i)\]

Donde: * \(\mathbf{x}\): Vector de cantidades consumidas de los \(n\) bienes. * \(x_i\): Cantidad consumida del bien \(i\). * \(\alpha_i\): Peso o preferencia del consumidor por el bien \(i\), con \(\alpha_i > 0\). * \(\ln(\cdot)\): Logaritmo natural.

Restricción Presupuestaria

La suma del gasto en todos los bienes no puede exceder el ingreso total disponible, \(I\).

\[\sum_{i=1}^n p_i x_i \leq I\]

Donde: * \(p_i\): Precio del bien \(i\). * \(I\): Ingreso total (renta). * La solución óptima siempre se encontrará en el límite, por lo que la restricción se trata como una igualdad.

2.1.2 El Problema de Optimización (Formal)

El problema formal del consumidor es:

\[\max_{\mathbf{x}} \; U(\mathbf{x}) = \sum_{i=1}^n \alpha_i \ln(x_i)\] \[\text{sujeto a } \sum_{i=1}^n p_i x_i \leq I\]

Este es un problema de optimización con restricción que se resuelve típicamente mediante el método de los multiplicadores de Lagrange.

2.1.3 Solución Derivada (Demanda Marshalliana)

La solución a este problema (la demanda Marshalliana para cada bien \(i\)) revela que la proporción del gasto en el bien \(i\) será igual al peso \(\alpha_i\) relativo, asumiendo \(\sum_{i=1}^n \alpha_i = 1\) para normalización (aunque la solución funciona para cualquier \(\alpha_i > 0\)).

\[\text{Gasto en bien } i: \quad p_i x_i^* = \frac{\alpha_i}{\sum_{j=1}^n \alpha_j} \cdot I\]

O la cantidad demandada óptima:

\[x_i^* = \frac{\alpha_i}{\sum_{j=1}^n \alpha_j} \cdot \frac{I}{p_i}\]

Esta formulación demuestra que la elasticidad-ingreso y la elasticidad-precio cruzada son unitarias para esta función de utilidad específica, lo que la hace un modelo simple y muy utilizado para ilustrar la teoría de la demanda.

import numpy as np
import matplotlib.pyplot as plt
from scipy.optimize import minimize

# ------------------------------
# Función de utilidad Cobb-Douglas (logarítmica)
# ------------------------------
def utilidad_cobb_douglas(x, precios, ingreso, alphas):
    if np.any(x <= 0):
        return -np.inf  # Penaliza valores negativos o cero (log no definido)
    gasto_total = np.dot(precios, x)
    if gasto_total > ingreso:
        return -np.inf  # Penaliza si excede el ingreso
    return np.sum(alphas * np.log(x))

# ------------------------------
# Parámetros del consumidor
# ------------------------------
n = 4
precios = np.array([2.0, 3.0, 1.5, 4.0])
ingreso = 100
alphas = np.array([0.25, 0.35, 0.20, 0.20])

# Condición: suma de alphas = 1 para interpretar como participación marginal
assert np.isclose(np.sum(alphas), 1), "Las α deben sumar 1"

# Punto inicial: consumo igualitario
x0 = np.ones(n) * (ingreso / n / np.mean(precios))

# ------------------------------
# Optimización
# ------------------------------
def objetivo(x):
    return -utilidad_cobb_douglas(x, precios, ingreso, alphas)

# Restricción: gasto ≤ ingreso
restriccion = {
    'type': 'eq',
    'fun': lambda x: ingreso - np.dot(precios, x)
}

# Límites: no se puede consumir cantidad negativa
limites = [(1e-6, None) for _ in range(n)]

resultado = minimize(objetivo, x0, method='SLSQP', bounds=limites, constraints=[restriccion])

# ------------------------------
# Resultados
# ------------------------------
x_opt = resultado.x
gasto_opt = precios * x_opt
utilidad_max = -resultado.fun
porcentajes_gasto = 100 * gasto_opt / ingreso

# ------------------------------
# Visualización
# ------------------------------
fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(15, 6))
bienes = [f"$x_{i+1}$" for i in range(n)]

# Gráfico 1: Consumo óptimo
bars = ax1.bar(bienes, x_opt, color='cornflowerblue', alpha=0.8)
ax1.set_title("Asignación Óptima del Consumidor (Cobb-Douglas)", fontsize=13)
ax1.set_ylabel("Cantidad Óptima")
ax1.grid(axis='y', alpha=0.3)

for bar, valor in zip(bars, x_opt):
    ax1.text(bar.get_x() + bar.get_width()/2, valor + 0.2, f"{valor:.2f}", ha='center', fontweight='bold')

# Gráfico 2: Distribución del gasto óptimo
ax2.pie(porcentajes_gasto, labels=bienes, autopct='%1.1f%%', startangle=90,
        colors=plt.cm.Set3.colors[:n], wedgeprops={'edgecolor': 'white'})
ax2.set_title("Distribución del Gasto Total", fontsize=13)

plt.tight_layout()
plt.show()

# ------------------------------
# Impresión de resultados
# ------------------------------
print("Resultados del Modelo de Optimización del Consumidor:")
print("-" * 50)
print(f"Utilidad máxima alcanzada: {utilidad_max:.4f}")
print(f"Ingreso utilizado: {np.sum(gasto_opt):.2f} de {ingreso}")
print(f"Composición óptima (cantidades): {x_opt.round(2)}")
print(f"Gasto por bien: {gasto_opt.round(2)}")
